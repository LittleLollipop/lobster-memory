# 长期记忆方案竞品对比：lobster-memory vs 主流方案

> **分析时间：2026 年 7 月 11 日**（竞品数据以该时点公开信息为准，后续可能变化）
> 目的：评估 lobster-memory（含 axolotl 图存储 + wb-lobster-memory 桥接）在 AI agent 长期记忆赛道中的真实水平，明确领先点与短板。
> 数据来源：Mem0 / Zep(Graphiti) / Letta(MemGPT) / Cognee 等公开仓库与基准报告（详见文末「数据来源」）。

---

## 一、当前长期记忆方案全景

2026 年该赛道已按「存储范式」分化清晰，主流玩家分四类：

| 范式 | 代表 | 一句话 |
|---|---|---|
| **混合 vector + graph + KV** | **Mem0**（~47K stars） | 最流行，自动抽取事实，5 行接入，生态最广 |
| **时态知识图** | **Zep · Graphiti** | 每条边带 `valid_from / invalid_at`，事实变更是「标记失效 + 新建」而非覆盖 |
| **分层 runtime 自编辑** | **Letta / MemGPT** | agent 自己用工具改 core memory block，完整 agent 运行时 |
| **全知识图 KM** | **Cognee** | 从文档/代码/邮件建图，偏知识管理 |
| 个人本地记忆 | Supermemory / Second Me / Personal.ai | 消费级、本地优先、非生产级 |
| 模型内置 | ChatGPT / Claude 记忆 | 向量式、黑盒、不可观测（与本方案反黑盒理念相反） |

## 二、逐维度对比（含我们）

| 维度 | Mem0 | Zep · Graphiti | Letta | **lobster-memory（我们）** |
|---|---|---|---|---|
| 存储模型 | 混合 vector+graph+KV | 时态知识图 | 分层 block + archival | **图（实体-关系-情绪 valence）** |
| 自动抽取 | ✅（LLM 抽取事实） | ✅ | ✅（agent 自编辑） | ✅（**抽取读记忆自成长=套娃**） |
| 时态版本化 | ❌ 覆盖式 | ✅ 有效性窗口 | ❌ | ❌ 覆盖式（**落后 Zep**） |
| 遗忘/巩固 | TTL / 时间衰减 | 时态失效保留审计 | 摘要下沉 | ✅ **显式 weight+时间剪枝，门控可查** |
| 情绪 / valence | ❌ | ❌ | ❌ | ✅ **显式情绪边 + 反馈 valence** |
| 可观察性 | ✅ 时间戳/版本/可导出 | ✅ 图可查 | 中 | ✅ **明文节点+边+因果边（caused/derived）** |
| 自成长性 | 营销词 "self-improving" | 无 | 无 | ✅ **递归抽取真闭环（竞品基本没有）** |
| 跨会话持久 | ✅ | ✅ | ✅ | ✅（已验证） |
| 生态/集成 | LangGraph/CrewAI/OpenAI/Anthropic | 同左 | LangGraph | ❌ **仅 WorkBuddy 桥接** |
| 自托管/开源 | ✅ Apache 2.0 | ✅ | ✅ | ✅（但单人维护） |
| 基准分数 | LongMemEval 自报 ~93.4（新算法） | 独立测 63.8（GPT-4o）领先 | ~52 | ❌ **未跑任何基准** |
| 成熟度/采用 | 高（47K stars，有托管云） | 中高 | 中高 | ❌ **早期原型，单人，未规模验证** |

> 注：Mem0 的 93.4 是其 2026-04 新算法自报数；独立博客用 GPT-4o 测得 Zep 63.8 vs Mem0 49.0。基准方法论不统一，数字别全信，但**趋势一致**：Zep 时态推理强，Mem0 靠新算法追平甚至反超，Letta 胜在长程自主。

## 三、我们的定位

**一句话：设计理念在第一梯队（甚至局部领先），工程成熟度和生态在最后梯队。是「想法很新、产品很嫩」的早期研究型原型。**

### 我们真站在前面（或独有）的地方
1. **递归自成长抽取（套娃）**：让「决定记什么」的抽取动作也读记忆，形成自指闭环。这是 Mem0/Zep/Letta **都没有**的机制——它们的抽取是一次性 LLM 调用，不涉及「记忆反哺抽取质量」。这是本系统真正的智力升级点。
2. **情绪 valence 作为一等公民**：把偏好/批评存成带 valence 的边，竞品基本不建模情绪维度。
3. **显式、可观察、安全的遗忘**：`consolidate` 有 `should --round` 门控、剪枝是选择性的。Zep 有时态失效、Mem0 有 TTL，但「遗忘是一门可核对的哲学」这点只有我们把它做成了可见旋钮。
4. **因果边（caused/derived）**：图里能存「他写 axolotl *因为* 恨黑盒」这种因果链，向量库和大多数图记忆只存关联不存因果。
5. **图里住着「如何维护图」的指令**：未来的抽取读得到今天的保养手册——回到递归自成长。

### 我们明显落后（甚至空白）的地方
1. **时态版本化**：硬伤。用户说「我 2020 年前就用 X」这种会变的事实，Zep 保留两个时间窗，我们只覆盖。在事实会过期的测试上大概率垫底。
2. **没跑过任何基准**：Mem0/Zep 都有 LongMemEval / DMR / BEAM 公开分数，我们连自测都没有——「水平」目前是主观判断，没法量化证明。
3. **成熟度与采用**：47K stars vs 我们单人单仓库；无多租户、无托管云、无企业合规（SOC2/HIPAA）、无 CrewAI/LangGraph 集成。
4. **生态绑定窄**：目前只有 WorkBuddy 桥接，存储层 axolotl 是自写 Rust 库，**未经过规模验证**——而 Mem0 后端是 Qdrant/Neo4j/pgvector 这类成熟件。
5. **检索延迟/运维**：Zep 自承图遍历 300–800ms、运维重；axolotl 未 benchmark，优劣未知。

---

## 四、结论与建议

**诚实定位：用正确哲学做了一个尚未产品化的原型。** 在「该往哪走」上比多数竞品想得深（反黑盒、可观察、遗忘促成长、递归自成长、情绪建模）；但在「能不能被别人用、扛不扛规模、过不过基准」上，连 Mem0 影子都没碰到。

若目标是「爬到第一梯队」，按性价比排最该补的三件事：

1. **补时态版本化**：给边加 `valid_from / invalid_at`，冲突时建新边而非覆盖——与 Zep 拉平的最短路径，且与现有图模型兼容。
2. **跑一次 LongMemEval**（哪怕只跑子集）：把「我们大概什么水平」从嘴上说变成有数。
3. **加一层标准集成**：OpenAI-compatible API + 一个 LangGraph / WorkBuddy 适配器——脱离「只有你能用」。

## 数据来源与备注

- Mem0：GitHub ~47K stars，Apache 2.0，混合 vector+graph+KV，自动事实抽取，LongMemEval 自报 ~93.4（2026-04 新算法）。
- Zep · Graphiti：时态知识图，每条边带有效性窗口；独立博客（GPT-4o）测 LongMemEval 63.8，领先 Mem0(49.0)。
- Letta / MemGPT：分层 runtime（core memory block + archival + recall），agent 自编辑记忆，偏长程自主。
- Cognee：从文档/代码/邮件构建全知识图，偏知识管理。
- 我们的方案实例规模（截至 2026-07-11 测试工作区）：约 41 节点 / 92 边 / 容量 0.3%，仍属早期苗态。
- 基准数字方法论不统一，横向比较仅作趋势参考，不构成精确排名。
